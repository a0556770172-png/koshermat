# כושרמט ♞ — פלטפורמת שחמט תחרותית

אתר מלא לשחמט מקוון: משחק בזמן אמת מול יריבים, דירוג ELO, דרגות, מדליות, צ'אט (בלובי ובתוך משחק), וּפאנל ניהול ופיקוח.

## מבנה הקבצים

```
index.html         דף נחיתה
auth.html          הרשמה / התחברות
lobby.html         לובי: שידוך יריבים, משחקים חיים, צ'אט קהילתי
game.html          עמוד המשחק (לוח, טיימרים, צ'אט, כניעה/תיקו/דיווח)
leaderboard.html   טבלת דירוג
profile.html       פרופיל אישי + עריכה
admin.html         פאנל ניהול (למנהלים בלבד)
css/style.css      כל העיצוב והאנימציות
js/                כל הלוגיקה (supabaseClient, ui, auth, lobby, game, leaderboard, profile, admin)
schema.sql         סכמת מסד הנתונים המלאה עבור Supabase
```

## שלב 1: הרצת מסד הנתונים ב-Supabase (חובה, פעם אחת)

1. היכנס ל-Dashboard של הפרויקט שלך בכתובת https://supabase.com/dashboard
2. לחץ על **SQL Editor** בתפריט הימני, ואז **New query**
3. פתח את הקובץ `schema.sql` מתוך תיקיית האתר, העתק את כל התוכן והדבק בעורך
4. לחץ **Run**. הפעולה יוצרת את כל הטבלאות, ההרשאות (RLS) והפונקציות הדרושות (שידוך יריבים, חישוב ELO, מדליות, חסימת משתמשים וכו')

> אפשר להריץ את הקובץ פעם אחת בלבד. אם תריץ אותו שוב בעתיד (לדוגמה אחרי עדכון), זה בטוח לרוב כי משתמשים ב-`if not exists` / `or replace`.

## שלב 2: הפיכת עצמך למנהל (כדי לגשת לפאנל הניהול)

1. הירשם לאתר פעם אחת דרך `auth.html`
2. ב-Supabase, לך ל-**Table Editor → profiles** ומצא את השורה שלך (לפי שם המשתמש), והעתק את ה-`id` שלה
   (או הרץ ב-SQL Editor: `select id, username from profiles;`)
3. הרץ ב-SQL Editor:
   ```sql
   update profiles set is_admin = true where id = 'ה-UUID-שהעתקת';
   ```
4. התחבר מחדש לאתר — לינק "🛡️ ניהול" יופיע בתפריט העליון

## שלב 3: פרסום האתר

האתר בנוי כ-HTML/CSS/JS סטטי לגמרי (ללא build step), כך שאפשר להעלות אותו כמו שהוא לכל שירות אחסון סטטי:

- **Netlify**: גרור את התיקייה לתוך https://app.netlify.com/drop
- **Vercel**: `vercel deploy` מתוך התיקייה, או חיבור לריפו ב-GitHub
- **GitHub Pages**: העלה את הקבצים לריפו והפעל Pages מהברירת מחדל (branch root)

אין צורך בשרת צד-שרת נפרד — כל התקשורת מתבצעת ישירות מול Supabase (מסד נתונים, אימות משתמשים וזמן-אמת).

## הערת אבטחה חשובה

בקוד האתר (`js/supabaseClient.js`) מוטמע רק **המפתח הפומבי** (`publishable/anon key`) — וזה תקין ובטוח, הוא מיועד לרוץ בדפדפן.

**המפתח הסודי (`secret/service_role key`) לא נכלל בשום קובץ באתר** ולא אמור להיכנס לקוד צד-לקוח בשום מצב — כל מי שמריץ "הצג מקור" בדפדפן היה רואה אותו ומקבל גישה מלאה למסד הנתונים, כולל עקיפת כל הרשאות האבטחה (RLS).

מכיוון שהמפתח הסודי כבר הודבק פעם אחת בשיחה, מומלץ בכל זאת:
1. להיכנס ל-Project Settings → API ב-Supabase
2. ללחוץ על **Rotate** ליד ה-secret key, כדי ליצור מפתח חדש שאיש לא ראה

## איך זה עובד מבפנים (בקצרה)

- **שידוך יריבים**: הצטרפות ל"תור" (`matchmaking_queue`) מפעילה טריגר בבסיס הנתונים שמחפש יריב מתאים ברמת דירוג קרובה ופותח משחק חדש אוטומטית.
- **דירוג ELO**: בסיום כל משחק (שח-מט / כניעה / נגמר הזמן / תיקו) מופעלת פונקציית `finish_game` שמחשבת דירוג חדש לשני הצדדים, מעדכנת ניקוד וסטטיסטיקות, ובודקת אם מגיעות מדליות חדשות.
- **זמן אמת**: מהלכים, הודעות צ'אט ורשימת המשחקים החיים מתעדכנים אצל כל המשתמשים מיידית דרך Supabase Realtime — בלי לרענן דף.
- **הגנה מפני רמאות**: שדות רגישים כמו דירוג, ניקוד והרשאת ניהול מוגנים בטריגר בבסיס הנתונים כך שרק פונקציות מאובטחות (לא המשתמש עצמו) יכולות לשנות אותם.

## הרחבות אפשריות להמשך

- טיימר "בליץ"/"קלאסי" לבחירה לפני המשחק (כרגע קבוע על 10 דקות לצד)
- הזמנת חבר למשחק פרטי בלינק ייעודי
- התראות Push כשמוצא יריב
- תמיכה בהתחברות עם Google/גוגל (OAuth) דרך Supabase Auth
